# FCS 速查卡 v1.0

> 一句话：一群人临时聚到一起做事的协作默认规则——简单、灵活、无需合同。

## 你适合用它吗

✅ 失业中，想找找机会
✅ 空闲中，有个想法
✅ 想找些志同道合的人一起干
✅ 项目还没公司化，几个人想一起做点事
✅ 成员多元（失业者 / 兼职 / 自由职业 / 有空闲者）
✅ 想要简单约定贡献和利益，不想吵架
❌ 已有正式股权结构 → 不适用
❌ 需要法律层面的合规 → 找律师

## 核心原则

- 职位不分高低，劳动时间等价：**1 小时 = 1 点**
- 贡献点永久归属，退出不清零
- 个人份额 = 个人点数 ÷ 团队总点数（动态）

## 贡献度量

| 类型 | 公式 |
|---|---|
| 劳动时间 | 1 小时 = 1 点 |
| 资金 | (金额 ÷ 当地小时工资) × 资金权重 |
| 生产材料 | (估值 ÷ 当地小时工资) × 资金权重 |

资金权重建议为 3。

## 关键流程

### 加入团队
- **初期**（未达约定人数）：发起人决定方式（邀请 / 自由报名）
- **足员后**：现有成员 >50% 同意

### 退出团队
- 随时可退
- 已有贡献点保留至兑现事件
- 未消耗的资金 / 生产材料按投资时金额带走
- 劳动时间投入不可回收

### 任务流程
- 任何人可发布（必填：描述、预估工时）
- 先认领先得
- 自由安排时间，赶截止日期前完成

### 验收流程
- 自我确认 或 角色（PM 等）确认
- 工时调整都被平台记录，有疑问直接沟通
- 沟通受阻 → 质疑机制 → 简单多数投票

### 决策流程（推荐）
1. 模块负责人决定
2. 项目经理 / 产品经理裁决
3. 简单多数投票

### 截止日期争议
发起人 → 产品经理 → 项目经理 → 多数共识 → 投票

## 投票门槛

| 事项 | 门槛 |
|---|---|
| 日常决策、加入、追加投资、质疑、兜底 | 简单多数 (>50%) |
| 修改合作愿景、欺诈认定 | 4/5 多数 |

## 利益分配

**兑现时**：个人所得 = 总利益 × (个人点数 ÷ 团队总点数)

**公司化时**：贡献点比例 = 股权比例

## 红线（千万别做）

- 伪造工时、虚报估值、盗用他人贡献 → **4/5 投票认定后所有点数清零**
- 不服从投票结果 → 自动退出（但已有点数保留）

## 知识产权

- 贡献的 IP / 版权仅限本项目使用
- 个人 IP 随个人退出而失效
- 想法可在实施前由成员认可后录入记录，作为后续争议参考
